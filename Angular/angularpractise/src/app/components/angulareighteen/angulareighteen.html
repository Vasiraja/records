9. Dependency Injection

Dependency Injection inject service into component

Create Service

Use in Component

inject

Without constructor we can use inject method to inject service into component

dataService = inject(DataService)

this.dataService.post()

Angular uses Dependency Injection (DI) to provide services automatically instead of creating them manually.

Services can exist at different levels in Angular. This is called Injector Hierarchy.

The hierarchy looks like this: Child Component → Parent Component → Root (Application level)

When Angular looks for a service, it always searches from bottom to top: Child → Parent → Root

Default Behavior

If a service is available in the child component, Angular will use that first.

If not found, it will move to parent, then finally to root.

Modifiers (Control the Search Behavior)

@Self(): Angular will look ONLY in the current component. It will not check parent or root.

@SkipSelf(): Angular will skip the current component and start searching from the parent.

@Optional(): If Angular cannot find the service, it will return null instead of throwing an error.

@Host(): Angular will search only up to the host component and will not go beyond that boundary.

Simple Understanding

Self → check only here if child has needed service only get there not go to parent

SkipSelf → skip here, check above like skip child service and go for parent service to take

Optional → don't throw error if not found, like if no service found just stop and dont cause error

Host → don't go outside boundary , if not found doesn't go beyond boundry

Angular DI works using a hierarchy, and modifiers help control where Angular searches for a service.

10. Zone and Zoneless

Initially Angular monitors change detection using zone.js

Now using signals, no need for zone-based monitoring. Signals handle reactive updates efficiently.

We can disable zone.js using: 

bootstrapApplication(AppComponent, {{ '{' }}
  providers: [
    provideZoneChangeDetection({{ '{' }} eventCoalescing: true {{ '}' }})
  ]
{{ '}' }});

After this, Angular stops auto checking everything. Only updates when signals change.

11. SSR and Hydration

SSR = rendering Angular app on server instead of browser

Hydration

Attaching Angular logic to already rendered HTML

Flow → server renders HTML → browser shows HTML → Angular hydrates (adds events, logic)

Enable using: ng add @angular/ssr

Config: 

bootstrapApplication(AppComponent, {{ '{' }}
  providers: [
    provideClientHydration()
  ]
{{ '}' }});

Selectors

i. Type Selector

type selector used as an HTML tag

<app-somecomponent></app-somecomponent>

selector: 'app-somecomponent'

other side : we can call <app-somecomponent></app-somecomponent>

ii. Attribute Selector

attribute selector used as an attribute on any element

<div app-somecomponent></div>

selector: '[app-somecomponent]'

other side : we can call <div app-somecomponent></div>

iii. Class Selector

class selector used as a css class on any element

<div class="app-somecomponent"></div>

selector: '.app-somecomponent'

other side : we can call <div class="app-somecomponent"></div>
